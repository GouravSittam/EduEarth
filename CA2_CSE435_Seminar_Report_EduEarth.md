## **1.1 Title of the Seminar Topic**

**EduEarth: Gamified Environmental Education Platform for Sustainable Learning**

## **1.2 Background and Importance of the Topic**

Environmental education is no longer optional in modern schooling. Climate change, biodiversity loss, pollution, and unsustainable consumption patterns are global challenges that require informed and responsible citizens. Traditional classroom pedagogy often delivers sustainability concepts in a theoretical way, which may reduce student engagement and practical understanding. At the same time, digital-native learners respond strongly to interactive and game-based learning experiences.

EduEarth is designed to address this gap by combining environmental education with gamification, experiential activities, progress analytics, and collaborative learning. The platform offers students interactive learning modules and quiz-driven gameplay, while teachers receive tools for content delivery, student monitoring, and performance evaluation.

The importance of this topic lies in the convergence of three major educational trends:

- Technology-enabled personalized learning

# **Chapter-2: Literature Review**

## **2.1 Introduction to Literature Context**

Educational technology literature highlights that interactive learning environments increase student participation and improve conceptual understanding when compared with lecture-only methods. Gamification elements such as badges, points, levels, and leaderboards improve motivation and persistence when implemented with clear learning goals.

In sustainability education, contextual relevance is crucial. Learners better retain concepts when they are connected to real-world environmental scenarios such as waste segregation, energy conservation, ecosystem health, and climate impacts.

## **2.2 Gamification in Education**

Research on gamified learning environments indicates that the following mechanics are most effective:

- Goal progression and visual feedback
- Immediate reward cycles for task completion
- Constructive competition through leaderboards
- Collaborative missions that encourage peer learning

EduEarth includes these mechanics in quiz attempts, reward points, achievements, and dashboard tracking. This alignment demonstrates practical application of theory-driven game mechanics.

## **2.3 Experiential and Constructivist Learning Models**

Experiential learning theory suggests that learners understand better through action, reflection, conceptualization, and re-application. Constructivist frameworks similarly argue that students build understanding actively rather than passively receiving information.

EduEarth follows this principle by:

- Presenting module content as contextualized learning units
- Evaluating understanding through quizzes
- Reinforcing progress with achievement systems
- Encouraging repeated participation through streak and reward tracking

## **2.4 Digital Platforms for Environmental Learning**

Existing online environmental platforms generally provide static informational content, video material, or standard MCQ assessments. Fewer systems combine full-stack role management, real-time learning engagement, adaptive reward structures, and explicit sustainability competency progression.

EduEarth contributes to this space with a multi-role architecture and integrated learning-game-assessment design.

## **2.5 Learning Analytics and Feedback Systems**

Recent literature emphasizes the role of educational analytics in improving outcomes. Analytics dashboards support early intervention, personalized support, and learner self-regulation. Metrics such as completion rates, quiz attempts, scores, and streaks can represent engagement quality.

EduEarth’s data model includes quiz attempts, user points, streaks, and achievement unlock timestamps, which are suitable indicators for analytics-driven pedagogy.

## **2.6 Synthesis of Literature Findings**

From the reviewed themes, effective sustainability learning systems should include:

1. Interactive content and active participation mechanisms.
2. Frequent low-stakes assessments with timely feedback.
3. Visible achievement progression and recognition.
4. Teacher-facing monitoring interfaces.
5. Scalable and secure backend infrastructure.

EduEarth maps strongly to these principles and provides a practical case for seminar-level technical and conceptual study.

# **Chapter-3: Conceptual Study / Seminar Work**

## **3.1 Overview of EduEarth System**

EduEarth is a full-stack educational platform designed for environmental learning. It integrates:

- **Frontend:** Next.js (React + TypeScript)
- **Backend:** Node.js + Express + TypeScript
- **Database:** PostgreSQL with Prisma ORM
- **Additional capabilities:** Socket-enabled interactions, role-based modules, and achievement tracking

The platform supports multiple stakeholders, each with dedicated interaction flows.

## **3.2 Core Concepts**

### **3.2.1 Role-Based User Management**

The system manages users with role-based access patterns. Core profile data includes login identity, password hash, role, streak metrics, and points. Parent-student mappings are captured to support supervised educational participation.

### **3.2.2 Modular Environmental Learning**

The `modules` entity organizes environmental topics with title, description, category, and level requirements. This supports structured curriculum progression.

### **3.2.3 Assessment Through Quizzes**

Each module can have one or more quizzes. Quiz entities include time limits and reward points. Questions are stored with options, correct answers, explanations, and type metadata.

### **3.2.4 Reward and Achievement System**

Achievements are designed to reward milestones. Unlock logic can be represented with condition codes and is persisted through user-achievement relationships.

### **3.2.5 Progress and Performance Tracking**

Quiz attempts capture score, completion status, and timestamp. Combined with user streak and points, this supports engagement analytics.

## **3.3 Database Model and Entity Relationships**

The system data model defines normalized relationships across users, modules, quizzes, questions, and achievements.

**Figure 3.1: EduEarth Database ER Diagram**

![Figure 3.1 - EduEarth Database ER Diagram](https://github.com/user-attachments/assets/4d9778b9-a778-4199-a32c-27ad9fce6df6)

### **3.3.1 Key Entity Descriptions**

- **users:** stores identity, authentication-linked metadata, points, streak, and timestamps.
- **user_relationships:** maps parent and student user IDs.
- **modules:** stores topic-level instructional units.
- **quizzes:** associates assessments with modules and reward policies.
- **questions:** stores question body, options, answer key, and explanation.
- **quiz_attempts:** records learning assessment outcomes.
- **achievements:** defines badges/milestones and unlock logic.
- **user_achievements:** stores achievement unlock instances by user.
- **environmental_data:** supports ingestion/storage of external sustainability datasets.

### **3.3.2 Relational Insights**

1. A module can map to multiple quizzes.
2. A quiz can map to multiple questions.
3. A user can perform multiple quiz attempts.
4. A user can unlock multiple achievements.
5. Parent-student mapping supports supervised learning contexts.

## **3.4 System Workflow / Conceptual Flow**

**Figure 3.2: EduEarth Conceptual/System Workflow Diagram**

![Figure 3.2 - EduEarth Workflow/Architecture Diagram](https://github.com/user-attachments/assets/ae45b195-f6c9-4fdf-a1dd-976173ecc5e8)

The conceptual workflow can be interpreted as:

1. User registers/authenticates and enters the role-specific interface.
2. Student explores modules by level/category.
3. Student attempts quizzes connected to selected modules.
4. Backend validates attempts and stores results.
5. Points/streaks/achievements are recalculated and persisted.
6. Dashboards display progress and analytics to users and educators.

## **3.5 Technical Architecture**

### **3.5.1 Frontend Layer**

Frontend implementation uses Next.js with component-driven architecture. Major capabilities include:

- Student dashboard views
- Teacher dashboard views
- Interactive game screens
- Authentication pages
- Course details and progress components

### **3.5.2 Backend Layer**

Backend services implement:

- Authentication and authorization APIs
- Institution/class/lesson/game routes
- Business logic for attempts and progression
- Data validation and persistence through Prisma

### **3.5.3 Data Layer**

The PostgreSQL schema, mapped via Prisma, enables integrity, consistency, and relationship modeling for educational workflows.
